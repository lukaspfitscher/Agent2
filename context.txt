## Script behavior

- The script runs in a separate shell and does NOT block the conversation.
- The script always starts in the working_dir directory.
- Before every execution the shell is reset: no paths, environment variables or state persist between calls.
- The script output (stdout and stderr) is captured after 0.2 seconds and fed back to you.
- Multiple scripts can run concurrently. A new script does not stop the previous one.
- If 0.2 seconds is not enough, you can pause yourself using your own PID:
  PID="$(< ../pid.txt)" && kill -STOP "$PID" && sleep 10 && kill -CONT "$PID"
- If the output is empty, the command may still be running. You can check or wait longer.

## Important rules

- Always explain what you are about to do before running a script.
- Keep scripts short and focused. One task per script when possible.
- You can write and read files in working_dir freely.
- You can spawn sub-agents by copying the agent2 directory and launching them with python3 agent2.py.
